## 公共组件实现说明

### 1. `types.h`

- 定义了仅面向基础设施的跨平台整数类型别名：
  - `NodeId`
  - `Term`
  - `LogIndex`
  - `QueueSize`
  - `Byte`
- 使用 `std::uint64_t`、`std::size_t`、`std::uint8_t`，避免平台相关宽度差异。
- 仅加入基础常量：
  - `kInvalidNodeId`
  - `kInitialTerm`
  - `kInvalidLogIndex`
- 未引入 `RaftRole`、`LogEntry`、`StoreInfo` 或任何业务结构。

### 2. `Status`

- 定义独立于 gRPC 的 `StatusCode` 与 `Status`。
- 支持以下状态码：
  - `OK`
  - `INVALID_ARGUMENT`
  - `NOT_FOUND`
  - `IO_ERROR`
  - `CORRUPTION`
  - `BUSY`
  - `RESOURCE_EXHAUSTED`
  - `RETRY_LATER`
  - `INTERNAL_ERROR`
- `Status` 支持：
  - 成功判断 `ok()`
  - 错误码访问 `code()`
  - 错误信息访问 `message()`
  - 字符串化 `ToString()`
- 提供静态构造函数，便于后续各模块统一返回结果。

### 3. `Checksum`

- 实现了纯标准库的确定性 64 位 FNV-1a 校验。
- 提供接口：
  - `Compute(const void*, size_t)`
  - `Compute(const std::string&)`
  - `Compute(const std::vector<uint8_t>&)`
  - 对应 `Verify(...)`
- 未定义 WAL 文件格式，仅提供可复用的字节校验能力。
- 未引入大型第三方依赖。

### 4. `BoundedQueue`

- 实现了基于 `std::mutex` 与 `std::condition_variable` 的线程安全有界队列模板。
- 提供能力：
  - `Push(const T&)`
  - `Push(T&&)`
  - `Pop(T*)`
  - `Close()`
  - `capacity()`
  - `size()`
  - `closed()`
- 行为约束：
  - 容量满时，`Push` 立即返回 `RESOURCE_EXHAUSTED`，不允许无限增长。
  - 容量为 0 时，`Push` 返回 `RESOURCE_EXHAUSTED`。
  - 队列关闭后，`Push` 返回 `BUSY`。
  - `Pop` 在空队列时等待；若等待期间队列关闭且仍为空，则返回 `RETRY_LATER` 退出等待。
- 未实现无锁队列、协程或平台专属线程 API。
