## ServerLifecycle 已实现的接口

- `Initialize(const config::Config& config)`
- `Start()`
- `IsRunning() const`
- `HasShutdownRequest() const`
- `RequestStop()`
- `Stop()`

行为说明：

- `Initialize` 初始化日志并注册 `ProcessOps` 关闭处理
- `Start` 只切换运行状态并记录当前活动生命周期对象
- `HasShutdownRequest` 同时检查内部停止标记和 `ProcessOps::IsShutdownRequested()`
- `RequestStop` 仅设置停止标记
- `Stop` 可重复调用，会清理运行状态并关闭日志
- 未创建业务线程、未监听网络、未构造 Raft/Storage/Runtime/RPC/Metadata/Store 对象

## Config、Logging 和 ProcessOps 的集成方式

- `ConfigLoader` 负责在外部先把 JSON 解析为 `Config`
- `ServerLifecycle` 只接收已解析的 `Config`，不会重新解析 JSON
- `ServerLifecycle::Initialize` 使用 `config.log_level` 调用 `InitializeLogging`
- `ServerLifecycle::Initialize` 调用 `platform::RegisterShutdownHandler(...)`
- `ServerLifecycle::HasShutdownRequest` 使用 `platform::IsShutdownRequested()`
- 关闭回调只设置停止标记，不执行复杂清理
