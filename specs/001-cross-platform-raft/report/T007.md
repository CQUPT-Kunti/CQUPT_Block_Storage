## 每组测试覆盖内容

### Status

- 成功状态默认值
- `INVALID_ARGUMENT` 错误码和错误信息
- `BUSY`、`RESOURCE_EXHAUSTED`、`RETRY_LATER` 状态码和字符串
- `NOT_FOUND`、`IO_ERROR`、`CORRUPTION`、`INTERNAL_ERROR` 工厂函数

### Checksum

- 相同内容产生相同校验值
- 不同内容产生不同校验值
- 空内容的稳定值
- 已知输入 `abc` 的稳定结果

### BoundedQueue

- 正常入队出队
- FIFO 顺序
- 队列满返回 `RESOURCE_EXHAUSTED`
- 零容量边界
- 关闭后 `Push` 返回 `BUSY`
- 关闭且空队列时 `Pop` 返回 `RETRY_LATER`
- `Pop(nullptr)` 返回 `INVALID_ARGUMENT`

### Config

- 正常配置可加载
- `node_id`、端口、空路径校验
- 当前节点必须存在于初始成员中
- `heartbeat_interval_ms < election_timeout_min_ms`
- `queue_capacity` 和 `max_message_size` 必须为正数
- 缺少字段、错误类型、非法 JSON 返回明确错误

说明：

- 测试配置在跨平台临时目录中即时生成
- 未修改现有配置 JSON

### FileOps

- 创建和存在性判断
- 创建或打开文件
- 写入后读取内容一致
- 文件大小正确
- 文件截断正确
- `Flush` 成功
- `Rename` 正确
- 删除文件正确
- `AtomicReplace` 后目标内容正确
- 不存在文件返回明确错误

说明：

- 使用 `std::filesystem::temp_directory_path()` 创建唯一测试目录
- 通过本地 RAII 临时目录夹具清理文件
- 未测试 WAL、Snapshot 或 FileRaftStorage
