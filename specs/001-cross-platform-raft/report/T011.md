## RaftCore Maintained State

`RaftCore` now maintains:

- current node id
- current role
- current term
- voted-for node id
- current leader id
- total logical tick count
- election elapsed tick count
- election timeout tick value
- one internal `HardState`
- one internal `RaftLog`

The implementation does not duplicate `RaftLog` internal log markers or entry storage outside the `RaftLog` object.

## Tick And Election Timeout Handling

- `Tick()` advances exactly one logical time unit per call
- `Follower` and `Candidate` increment election elapsed ticks
- `Leader` does not start a new election on tick
- `Learner` does not start a new election on tick
- when election elapsed ticks reach the configured election timeout:
  - `Follower` or `Candidate` starts a new election
  - term increments
  - self vote is recorded
  - known leader id is cleared
  - election elapsed ticks reset
- `Tick()` returns a minimal action enum:
  - `NONE`
  - `START_ELECTION`

No real time, `std::chrono`, sleep, threads, or randomness are used.

## Role Transition Rules

Implemented transitions:

- `BecomeFollower(term, leader_id)`
- `StartElection()`
- `BecomeLeader()`
- `UpdateElectionTimeout(ticks)`

Rules enforced:

- election timeout must be greater than zero
- term cannot move backward
- entering a higher term clears old vote
- starting an election increments term
- starting an election sets `voted_for` to current node id
- starting an election clears known leader id
- only `Candidate` can become `Leader`
- `Leader` cannot start a new election
- `Learner` cannot become `Candidate`
- `Learner` cannot become `Leader`
- `Learner` cannot transition to `Follower` through the current public conversion API
- failed transitions return error before mutating state

## Term And Voted-For Updates

- `current_term` and `voted_for` are stored in `HardState`
- initialization preserves provided `HardState`
- `StartElection()`:
  - increments `current_term`
  - sets `voted_for = node_id`
- `BecomeFollower()` with higher term:
  - updates `current_term`
  - clears `voted_for`
- `BecomeFollower()` with same term:
  - keeps current vote

## Learner Restriction

- learner ticks do not trigger elections
- learner cannot call `StartElection()`
- learner cannot call `BecomeLeader()`
- learner cannot call `BecomeFollower()` in this minimal T011 API
