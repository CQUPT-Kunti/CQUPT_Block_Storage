## Snapshot File And Format

- Snapshot file name: `snapshot.bin`.
- Encoding: fixed-width little-endian integers plus raw bytes.
- No C++ struct memory, padding, JSON, Protobuf, or third-party serialization is used.

Snapshot payload layout:

```text
uint32 format_version
uint64 last_included_index
uint64 last_included_term
uint64 membership_configuration_id
uint32 has_active_transition
uint64 voter_count
repeat voter_count:
  uint64 node_id
  uint32 port
  uint64 host_length
  byte[host_length] host
uint64 learner_count
repeat learner_count:
  uint64 node_id
  uint32 port
  uint64 host_length
  byte[host_length] host
uint64 payload_length
byte[payload_length] payload
uint64 checksum
```

- Checksum covers every preceding Snapshot byte, excluding the checksum field itself.
- Decode validates version, index, term, member count, member id, port range, host length, payload length, truncation, trailing bytes, and checksum.
- Payload remains opaque and business independent.

## Save Flow

- `FileRaftStorage::SaveSnapshot()` validates:
  - snapshot index and term are positive
  - snapshot boundary does not move backward
  - snapshot index does not exceed committed index
  - snapshot term matches the existing log or current snapshot boundary
- Snapshot bytes are written through `platform::AtomicReplace(snapshot.bin, bytes)`.
- `AtomicReplace` uses a same-directory temporary file, flushes through FileOps, then replaces the target.
- After Snapshot replacement succeeds, `log.bin` is rewritten through `AtomicReplace` with only entries after `last_included_index`.
- Memory state is updated only after Snapshot and compacted log replacement both succeed.

Safety note:

- Snapshot is written before log compaction. If compaction rewrite fails after Snapshot replacement, restart recovery can still filter old covered log entries using the newer Snapshot boundary.
- The implementation never deletes the old Snapshot before writing a replacement.

## Load And Recovery

`Open()` and `Load()` now recover in this order:

- Read optional `snapshot.bin`.
- Decode and verify Snapshot if present.
- Read `hard_state.bin`.
- Read `log.bin`.
- Ignore records at or before Snapshot boundary.
- Require remaining logs to start at `snapshot_index + 1` and stay contiguous.
- Check `commit_index <= last available index`.
- Check `commit_index >= snapshot_index` when Snapshot exists.
- Return Snapshot metadata and payload through `RaftStorageLoadResult::snapshot`.

## Log Tail Handling

- Incomplete log file tail is detected during record decoding.
- Complete records before the incomplete tail are kept.
- The repaired log is rewritten through `platform::AtomicReplace`, so repair uses FileOps flush and replace.
- A complete record with checksum mismatch is `Status::Corruption` and is not ignored.
- Snapshot corruption, unsupported version, illegal length, or truncation is `Status::Corruption` and is not ignored.
