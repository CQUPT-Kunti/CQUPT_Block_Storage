## Runtime Event Types

Defined in `raft_runtime.h` as simple structs aggregated by `std::variant<...>`:

| Event Struct | Fields | Purpose |
|---|---|---|
| `TickEvent` | (empty) | Logical tick trigger |
| `ShutdownEvent` | (empty) | Graceful shutdown request |
| `MessageEvent` | `source_node_id`, `message` (RaftMessage) | Inbound Raft network message |
| `ProposalEvent` | `proposal_id`, `payload` (OpaquePayload) | Client proposal |
| `MembershipChangeEvent` | `added_voters`, `added_learners`, `removed_nodes` | Membership configuration change |
| `PersistenceCompletionEvent` | `status`, `has_hard_state`, `hard_state`, `last_log_index`, `has_snapshot`, `snapshot` | Storage operation result |
| `ApplyCompletionEvent` | `applied_index`, `status` | State-machine apply result |

The shared variant type `RuntimeEvent` = `std::variant<TickEvent, MessageEvent, ...>` avoids virtual dispatch and class hierarchies.

All events use value semantics (no raw pointers). No gRPC, Protobuf, Metadata, or business types appear in the event structures.

## Bounded Queue Capacity and Backpressure

- **Queue type**: `common::BoundedQueue<RuntimeEvent>` (T003 implementation, thread-safe with `std::mutex` + `std::condition_variable`)
- **Capacity**: set via `Options::event_queue_capacity`, must be greater than zero
- **Full queue**: returns `Status::ResourceExhausted("queue is full")`
- **Closed queue**: returns `Status::Busy("queue is closed")` (from BoundedQueue after `Close()`)
- **Zero capacity**: returns `Status::ResourceExhausted("queue capacity is zero")` (from BoundedQueue)
- No events are silently dropped

## Enqueue Interfaces

All enqueue methods accept both `const&` and `&&` overloads where appropriate:

| Method | Input | Returns |
|---|---|---|
| `EnqueueTick()` | — | `Status` |
| `EnqueueMessage(source, message)` | `NodeId` + `RaftMessage` (copy or move) | `Status` |
| `EnqueueProposal(event)` | `ProposalEvent` (copy or move) | `Status` |
| `EnqueueMembershipChange(event)` | `MembershipChangeEvent` (copy or move) | `Status` |
| `EnqueuePersistenceCompletion(event)` | `PersistenceCompletionEvent` (copy or move) | `Status` |
| `EnqueueApplyCompletion(event)` | `ApplyCompletionEvent` (copy or move) | `Status` |
| `RequestShutdown()` | — | `Status` |

## Proposal Enqueue Semantics

- `EnqueueProposal()` adds the proposal to the queue and returns immediately.
- A successful return does **not** mean the proposal has been committed or applied.
- The `ProposalEvent` contains only a `proposal_id` and an `OpaquePayload` — no business command types are introduced.
- Final client success requires commit and apply (T023 scope).

## Persistence Completion and Apply Completion Data Structures

### PersistenceCompletionEvent

Mirrors the T019 `RaftStorageOperationResult` contract:

- `status`: OK on success, error otherwise
- `has_hard_state` / `hard_state`: the persisted HardState
- `last_log_index`: the last persistently written log index
- `has_snapshot` / `snapshot`: optional Snapshot boundary metadata

Failure results are preserved — never converted to success.

### ApplyCompletionEvent

- `applied_index`: the log index that was applied
- `status`: application result (OK or error)
- No state-machine call or `appliedIndex` advancement — those belong to T023.

## Shutdown Behavior

- `RequestShutdown()` enqueues a `ShutdownEvent` and sets `shutdown_requested_` to `true`.
- After shutdown is requested, subsequent ordinary events (Tick, Message, Proposal, etc.) return `Status::Busy("runtime is shutting down")`.
- Repeated `RequestShutdown()` calls return `Status::Busy("shutdown already requested")`.
- `shutdown_requested()` returns `true` after a successful shutdown request.
- `accepting_events()` returns `true` only when shutdown has not been requested.
- Queue draining and thread joining are not implemented (T024 scope).

## Runtime State

- `shutdown_requested_`: `bool`
- `event_queue_`: `BoundedQueue<RuntimeEvent>` — provides `capacity()`, `size()`, `closed()`.
- Query methods: `shutdown_requested()`, `accepting_events()`, `event_queue_size()`, `event_queue_capacity()`.
- No thread, protocol loop, persistence worker, apply worker, or peer outbound queue was added.
