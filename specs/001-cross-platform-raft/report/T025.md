## Production Code Fix: ProposalResult Overflow Tracking

**Problem**: `EmitProposalResult()` silently dropped results when the bounded `ProposalResult` queue was full. No indicator was available to detect data loss.

**Fix**: Added an `overflow` flag to `ResultQueue`. When a push fails due to full capacity, `overflow` is set to `true`. Two public query methods were added:

- `HasProposalResultOverflow()` — returns true if drops occurred since last clear
- `ClearProposalResultOverflow()` — resets the flag

This is a minimal fix — no Promise/Future/RPC context was introduced.
