## Temporary gRPC Server Assembly

Each integration test creates a private loopback server in test code only:

1. Build a `RaftCore` + `MemoryRaftStorage` + `RaftRuntime`
2. Construct the existing `RaftRpcServiceAdapter`
3. Start a temporary `grpc::Server` through `grpc::ServerBuilder`
4. Bind to `127.0.0.1:0`
5. Read back the system-assigned port
6. Initialize `GrpcRaftTransport` with that loopback address

The server is never promoted into production code and no shared server framework was added.

## Transport, RPCAdapter, and Runtime Connection Path

The end-to-end path exercised by the new tests is:

1. `GrpcRaftTransport::Send()`
2. real gRPC unary call
3. temporary `RaftRpcServiceAdapter`
4. `RaftRuntime::EnqueueMessage()`
5. runtime protocol processing
6. runtime peer response queue
7. adapter response conversion
8. gRPC unary response
9. transport response conversion back to `RaftMessage`

This is a real network loopback call, not a direct adapter invocation.

## Business Error and Transport Error Verification

Business rejection path:

- stale-term `RequestVote` returns `Status::OK()` from transport
- response carries `vote_granted = false`
- reject reason remains `STALE_TERM`

Transport / infrastructure failures:

- unknown target -> `NotFound`
- transport not started -> `Busy`
- transport stopped -> `Busy`
- runtime not running behind the adapter -> non-OK status with clear message
- stopped server -> `RetryLater`
- adapter-side response timeout -> `RetryLater`

This confirms business rejection and transport failure remain distinct.

## Lifecycle and Resource Cleanup

Verified in tests:

- startup order: runtime -> adapter -> temporary server -> transport
- normal stop of transport
- repeated transport `Stop()` is safe
- repeated test-server stop helper is safe
- runtime shutdown occurs after each test
- no fixed listener remains because each test uses its own ephemeral port and local server instance

Cleanup behavior:

- test fixture `TearDown()` stops transport first
- then shuts down and waits for the temporary gRPC server
- then requests runtime shutdown and waits for thread joins

No production listener, process lifecycle manager, or server wrapper was added.
